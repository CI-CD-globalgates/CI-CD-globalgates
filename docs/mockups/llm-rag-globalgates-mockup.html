Priority

적용 우선순위

지금 프로젝트엔 “재밌는 데모”보다 운영 데이터에 붙는 AI가 필요하다.

1st 견적 요청 작성 비서

이 서비스의 핵심은 좋은 질문을 보내는 것이다. 견적 본문, 태그, 누락 정보, 대상 전문가를 함께 정리해 전환율을 바로 건드린다.

2nd 전문가 연결 어시스턴트

추천 자체보다 추천 이유가 중요하다. 전문가 카드, 연결 근거, 문의 초안을 한 흐름으로 묶으면 실제 행동으로 이어진다.

3rd 화상회의 후속조치 비서

이미 녹음과 요약 UI가 있으니, 회의 후 해야 할 일과 후속 메시지를 자동 정리해 차별점을 만드는 편이 낫다.

4th 무역 실무 탐색 비서

뉴스, 커뮤니티, 전문가, 상품을 근거로 묶어 답하는 탐색 비서는 분명 유용하지만, 거래 직전 기능보다는 한 단계 뒤다.

5th 상품 등록 품질 보조

카테고리 추천 계획이 이미 있으니, 스펙 보완과 태그 제안까지 붙이면 등록 품질을 높일 수 있다.

Architecture

권장 아키텍처

이 프로젝트는 검색 서비스가 아니라 거래 워크플로우 서비스다. 그래서 답변 생성보다 다음 행동 연결이 중요하다.

1. Source Layer

상품, 뉴스, 전문가 프로필, 커뮤니티, 견적, 채팅/회의 요약, 팔로우 그래프를 별도 인덱스로 유지

2. Intent Router

질문이 탐색인지, 견적 작성인지, 전문가 연결인지, 회의 후속조치인지 먼저 분기

3. Retrieval + Rerank

keyword + vector + metadata filter를 같이 쓰고, 최신성, 활동성, 전환 이력으로 재정렬

4. Action Copilot

답변보다 견적 초안, 추천 이유, 후속 메시지, 체크리스트 같은 행동 산출물을 먼저 노출

5. Feedback Loop

전문가 선택, 견적 전송, 회의 후속 메시지 발송, 카테고리 채택 여부를 다시 학습 데이터로 저장

추천 방향: 견적 작성 → 전문가 연결 → 회의 후속조치 → 실무 탐색 → 상품 등록 품질 보조
